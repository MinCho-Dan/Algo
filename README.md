# 📝 Algorithm Problem Solving (PS)

알고리즘 실력 향상을 위해 꾸준히 문제를 풀고 기록하는 저장소입니다.  
문제를 풀며 배운 전략과 효율적인 코드 작성법을 정리합니다.

## 📋 나만의 커밋 컨벤션 (Commit Convention)
일관된 커밋 메시지를 통해 변경 사항을 쉽게 확인합니다.

* **Format:** `[플랫폼] 문제번호_문제이름 (언어)`
* **Example:** `[BOJ] 1260_DFS와BFS (Java)`
* **Type:**
    * `Solve`: 문제 해결 시
    * `Fix`: 기존 코드 수정 또는 최적화 시
    * `Refactor`: 코드 구조 개선 시

---

## 💡 문제 풀이 원칙
1. **시간 제한 준수:** 실제 코딩 테스트처럼 시간을 정해놓고 풀이합니다.
2. **직접 구현:** 정답을 보기 전 최소 1시간은 스스로 고민합니다.
3. **복습:** 풀리지 않았던 문제는 일주일 뒤에 다시 풀어봅니다.
4. **최적화:** 문제를 푼 후 다른 사람의 풀이를 참고하여 메모리나 실행 시간을 최적화합니다.
